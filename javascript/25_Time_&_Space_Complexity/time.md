_________________________________  Time Complexity  ________________________________________________________

“Time complexity tells us how the running time of an algorithm grows as the input size increases.”

“Time complexity tells us how fast or slow an algorithm is based on the input size.”

“Time complexity helps us analyze the efficiency of an algorithm for large inputs.”



Time Complexity Of Different 

1] Linear Search

Best Case    → O(1)
Average Case → O(n)
Worst Case   → O(n)

2] Binary Search

Best Case    → O(1)
Average Case → O(log n)
Worst Case   → O(log n)

3] Sorting

i) Bubble Sort

Best Case    → O(n)
Average Case → O(n²)
Worst Case   → O(n²)

ii) Selection Sort

Best Case    → O(n²)
Average Case → O(n²)
Worst Case   → O(n²)

iii) Insertion Sort

Best Case    → O(n)
Average Case → O(n²)
Worst Case   → O(n²)


4] Merge Sort & Quick Sort

i) Merge Sort

Best Case    → O(n log n)
Average Case → O(n log n)
Worst Case   → O(n log n)

ii) Quick Sort

Best Case    → O(n log n)
Average Case → O(n log n)
Worst Case   → O(n²)

5] HashMap / HashSet Operations

Insert
O(1)

Delete
O(1)

Search
O(1)

Worst case:  (due to collisions)
O(n)  

6] Nested Loop Pattern

Two Nested Loops
O(n²)

Three Nested Loops
O(n³)
